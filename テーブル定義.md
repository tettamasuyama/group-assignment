# テーブル定義 (社員管理アプリ)

## users　テーブル

### 1.1 概要
ユーザーを管理するテーブル。
認証および権限管理に使用する。

### 1.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|----------|----|------|----|--------|------|
| id | BIGSERIAL | ○ | ○ | ○ | ユーザーID |
| email | VARCHAR(254) | ○ | - | ○ | ログインメール |
| password | VARCHAR(60) | ○ | - | - | パスワード（ハッシュ） |
| role | VARCHAR(20) | ○ | - | - | 権限 |

### 1.3 制約

- PRIMARY KEY: id
- UNIQUE: users.email
- NOT NULL: id
- NOT NULL: email
- NOT NULL: password
- NOT NULL: role


### 1.4 値制約
- role: ADMIN | EMPLOYEE

### 1.5 CRUD仕様
- CREATE: 新規ユーザー登録時
- READ: (現時点では使用予定なし)
- UPDATE: (現時点では使用予定なし)
- DELETE: 物理削除(現時点では実装予定なし)


## 2. employees　テーブル

### 2.1 概要

社員情報を管理するテーブル。
CSVインポートおよび管理画面から登録・更新される。

### 2.2 カラム定義

| カラム名 | 型 | 必須 | PK | UNIQUE | 説明 |
|----------|----|------|----|--------|------|
| id | BIGSERIAL | ○ | ○ | ○ | 社員ID | 
| employee_number | VARCHAR(20) | ○ | - | ○ | 社員番号 |
| name | VARCHAR(50) | ○ | - | - | 氏名 |
| email | VARCHAR(254) | ○ | - | ○ | メール |
| department | VARCHAR(50) | ○ | - | - | 部署 |
| position | VARCHAR(50) | ○ | - | - | 役職 |
| hire_date | DATE | ○ | - | - | 入社日 |
| status | VARCHAR(20) | ○ | - | - | 在籍状態 ||

### 2.3 制約

- PRIMARY KEY: id
- UNIQUE: employees.employee_number
- UNIQUE: employees.email
- NOT NULL: employee_number
- NOT NULL: name
- NOT NULL: email
- NOT NULL: department
- NOT NULL: position
- NOT NULL: hire_date
- NOT NULL: status

### 2.4 値制約
- department: GENERAL(総務) | HR(人事) | PLANNING（企画）| SALES（営業） 
- position : STAFF(一般社員) | LEADER(主任) | MANAGER(課長) | DIRECTOR(部長) | EXECUTIVE(本部長) | CEO(社長)
- status: ACTIVE | RETIRED

### 2.5 CRUD仕様
- CREATE: 管理者が登録、またはCSVインポート
- READ: ADMIN → 全件取得 | EMPLOYEE → 一部カラムのみ取得（name, email）
- UPDATE: 管理者のみ可能
- DELETE: 物理削除

### 2.6 更新ルール
- employee_number が一致する場合は更新（UPSERT）
- CSVインポート時は一括処理
