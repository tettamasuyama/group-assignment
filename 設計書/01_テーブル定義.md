# テーブル定義（社員管理アプリ）

## 1. users テーブル

### 1.1 概要

ログインユーザーを管理するテーブル。
認証および権限管理に使用する。

### 1.2 カラム定義

| カラム名     | 型            | 必須 | PK | UNIQUE | 説明           |
| -------- | ------------ | -- | -- | ------ | ------------ |
| id       | BIGSERIAL    | ○  | ○  | -      | ユーザーID       |
| email    | VARCHAR(254) | ○  | -  | ○      | ログインメールアドレス  |
| password | VARCHAR(60)  | ○  | -  | -      | ハッシュ化済みパスワード |
| role     | VARCHAR(20)  | ○  | -  | -      | 権限           |

### 1.3 制約

* PRIMARY KEY: id
* UNIQUE: email
* NOT NULL: email
* NOT NULL: password
* NOT NULL: role

### 1.4 値制約

* role: ADMIN / EMPLOYEE

### 1.5 CRUD仕様

* CREATE: 新規ユーザー登録時
* READ: ログイン認証時、JWT認証時に使用
* UPDATE: 現時点では実装予定なし
* DELETE: 社員削除時に、対応するユーザー情報を物理削除する場合がある

## 2. employees テーブル

### 2.1 概要

社員情報を管理するテーブル。
管理者画面からの登録・更新・削除、およびCSVインポートで使用する。

### 2.2 カラム定義

| カラム名            | 型            | 必須 | PK | UNIQUE | 説明      |
| --------------- | ------------ | -- | -- | ------ | ------- |
| id              | BIGSERIAL    | ○  | ○  | -      | 社員ID    |
| employee_number | VARCHAR(20)  | ○  | -  | ○      | 社員番号    |
| name            | VARCHAR(50)  | ○  | -  | -      | 氏名      |
| email           | VARCHAR(254) | ○  | -  | ○      | メールアドレス |
| department      | VARCHAR(50)  | ○  | -  | -      | 部署      |
| position        | VARCHAR(50)  | ○  | -  | -      | 役職      |
| hire_date       | DATE         | ○  | -  | -      | 入社日     |
| status          | VARCHAR(20)  | ○  | -  | -      | 在籍状態    |

### 2.3 制約

* PRIMARY KEY: id
* UNIQUE: employee_number
* UNIQUE: email
* NOT NULL: employee_number
* NOT NULL: name
* NOT NULL: email
* NOT NULL: department
* NOT NULL: position
* NOT NULL: hire_date
* NOT NULL: status

### 2.4 値制約

* department: GENERAL（総務） / HR（人事） / PLANNING（企画） / SALES（営業）
* position: STAFF（一般社員） / LEADER（主任） / MANAGER（課長） / DIRECTOR（部長） / EXECUTIVE（本部長） / CEO（社長）
* status: ACTIVE（在籍） / RETIRED（退職）

### 2.5 CRUD仕様

* CREATE: 管理者による社員登録、またはCSVインポート
* READ:

  * ADMIN: 社員情報を全件取得
  * EMPLOYEE: 自分の社員情報、または一部カラムのみ取得
* UPDATE:

  * ADMIN: 社員情報の更新が可能
  * EMPLOYEE: 自分の氏名・メールアドレスのみ更新可能
* DELETE: 管理者のみ物理削除

### 2.6 更新ルール

* employee_number が一致する場合は既存社員情報を更新する
* employee_number が存在しない場合は新規登録する
* CSVインポート時も employee_number を基準に登録・更新を判定する
